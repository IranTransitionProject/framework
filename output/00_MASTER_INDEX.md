## HOW TO USE THIS SYSTEM

This project has been restructured from monolithic documents into self-contained modules. **This index is the only file you must read first.** It tells you which modules to pull for any question.

**Rules:**
1. Every session begins by reading this index + `00_INSTRUCTIONS.md`.
2. Load only the modules relevant to the session's task.
3. Each module is self-contained with its own epistemic tags and source log.
4. Cross-references use module codes (e.g., "see ITB-A10 §3.4").
5. Updates happen at the module level — updating A10 does not require touching A1.

**Governance Protocol (applies to all modules):**
1. **Retention Rule:** DO NOT DELETE. No section, variable, or citation may be removed.
2. **Update Logic:** New data added as diffs. Contradictions marked [CONFLICT].
3. **Epistemic Tags:** [Fact], [Inference], [Uncertain], [Speculation].
4. **Confidence Bands:** [High] multi-source/audited; [Med] single institutional/media consensus; [Low] leak-based/single-source/extrapolated.
5. **BOM Discipline:** Institutional anatomy tagged with BOM markers (Level 0/1/2).
6. **Diaspora Bias Check:** Internal sources over external. [DIASPORA SOURCE] tag when needed.
7. **Factional Neutrality:** "What must happen regardless of who governs."
8. **OPSEC:** Analytical frameworks safe; operational details not.
9. **Regime Source Filtering:** Iranian state media treated as adversarial. Cited for claims, not facts.
